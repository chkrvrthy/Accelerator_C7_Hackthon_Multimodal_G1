Multimodal AI Design Analysis Suite

Interactive walkthrough · Accelerator C7 Hackathon

5 specialist AI agents review your UI design — in parallel.

Upload 1 to 5 screenshots of the same product. A LangGraph orchestrator fans out to Visual, UX, Accessibility, Brand-Consistency, and Market-Research agents — each vision agent sees every frame in one call, so the team treats the upload as one coherent product. Image RAG (CLIP + LanceDB) queries every frame and dedupes against an in-house corpus, surfacing which screen surfaced each ref. Deterministic pre-tools (k-means palette, OpenCV text-size, CTA density, Δ-E palette distance) feed each agent <measured_facts> so the LLM never has to invent numbers. A synthesizer aggregates everything into one prioritized DesignReport with per_frame_scores heatmap and affected_frames attribution, gated by a pure-Python quality check that re-prompts once on regressions. Same flow callable from any MCP-compatible coding agent.

Step-by-step (click any step to expand)

  1. 1 User drops dashboard.png and clicks Run Person E ▼

    The Gradio file uploader hands a list of 1..5 paths to on_run, plus an optional newline-separated frame_labels_text. Before any tokens are spent, we run preflight_batch: validates count (≤ 5), suffix, file size (≤ 20 MB each), and pixel count, then downsizes each frame to 1024 px on the long edge. UploadError is the only non-Exception path; everything else gets wrapped in a wide try/except that yields a friendly banner via _classify_run_error.

    ui/handlers.py
    def on_run(image, instructions, frame_labels_text=""):
    # 1. Image safety gate — preflight + auto-resize before any LLM call.
    raw_paths = _normalize_uploads(image)              # accepts list or single
    try:
        preflight_batch(raw_paths)                     # caps at 5, per-file checks
        image_paths = [downsize_for_pipeline(p) for p in raw_paths]
    except UploadError as e:
        yield (_status_message(e.user_title, e.user_body), {}, None); return

    labels = _resolve_frame_labels(frame_labels_text, image_paths)  # padded with stems
    use_real = _default_real_mode()
    deps = build_default_deps(use_real=use_real)
    get_cost_tracker().reset()
    try:
        report = run_graph(image_paths, instructions=instructions or None,
                           frame_labels=labels, deps=deps)
        yield (_status_message("Report ready", ...), report.model_dump(), report.model_dump())
    except Exception as e:
        title, body = _classify_run_error(e)         # CircuitOpenError -> friendly banner
        yield (_status_message(title, body), {}, None)
  2. 2 UI calls run_graph(...) E → A ▼

    The UI hands off to Person A's orchestrator. From here, all logic runs in the graph. Single-frame and multi-frame both flow through the same entry point — the orchestrator just looks at len(image_paths).

    src/agents/graph.py
    def run_graph(image_paths, *, instructions=None, frame_labels=None,
              deps=None) -> DesignReport:
    deps = deps or build_default_deps()
    paths = ([str(image_paths)] if isinstance(image_paths, (str, Path))
             else [str(p) for p in image_paths])
    state = GraphState(image_paths=paths,
                       frame_labels=list(frame_labels) if frame_labels else [],
                       instructions=instructions)
    # GraphState validator pads frame_labels with filename stems and
    # mirrors image_path = image_paths[0] for legacy single-image pre-tools.
    compiled = build_graph(deps)
    final = compiled.invoke(state)
    return final["report"]

    src/rag/retriever.py
    async def retrieve_by_image(self, image_path, k=5) -> list[RetrievedRef]:
    vec = self.embedder.embed_image(image_path)         # 512-d CLIP
    rows = self.store.query_by_vector(vec, k=k)         # LanceDB k-NN
    return [self._to_ref(r) for r in rows if (1 - r["_distance"]) > 0.2]

    async def search(self, query: str, k: int = 5) -> list[SearchResult]:
    raw = await self._client.search(query, max_results=k, include_answer=False)
    return [SearchResult(title=r["title"], url=r["url"], snippet=r["content"])
            for r in raw["results"]]
  13. 9 Each agent calls the LLM via OpenRouter Person A ▼

    OpenAI SDK against https://openrouter.ai/api/v1 with response_format={"type":"json_schema",...}. The cost tracker logs every call; the circuit breaker fast-fails after a small burst of 5xx/429s. System prompts come from the src/utils/prompts/ package and carry the shared ANTI_HALLUCINATION_RULE + ABSTENTION_RULE. Returns a validated Pydantic instance.

    src/llm/openrouter_client.py
    @cached
@_with_circuit_breaker
async def complete(self, *, system, user, schema, model=None, temperature=None):
    resp = await self._client.chat.completions.create(
        model=model or settings.default_text_model,
        messages=[{"role":"system","content":system},      # carries ANTI_HALLUCINATION + ABSTENTION rules
                  {"role":"user","content":user}],
        response_format={"type":"json_schema","json_schema":{
            "name": schema.__name__, "schema": schema.model_json_schema()}},
        temperature=temperature or settings.default_temperature)
    get_cost_tracker().record(resp.usage)               # tokens + USD ledger for the Settings tab
    return schema.model_validate_json(resp.choices[0].message.content)
  14. 10 Cost cache may short-circuit any LLM call Person A ▼

    Disk-backed JSON cache. The same screenshot returns instantly on the second run — demos are free.

    src/llm/cost.py
    def cached(fn):
    async def wrapper(self, **kw):
        key = prompt_hash(**kw, schema_name=kw["schema"].__name__,
                          model=kw.get("model") or settings.default_text_model)
        if (hit := cache.get(key)) is not None:
            return kw["schema"].model_validate(hit)
        result = await fn(self, **kw)
        cache.put(key, result.model_dump())
        return result
    return wrapper
  15. 11 Synthesizer aggregates all 5 outputs into one DesignReport Person A ▼

    The synthesizer aggregates, then a pure-Python quality_gate checks for known regressions (empty top_recommendations, ratings collapsed to 50.0, missing rationales). On a fail-severity violation the synthesizer is re-prompted once with the violations appended to its user message. We intentionally cap retries at 1 for cost discipline.

    src/agents/synthesizer.py
    async def run(state: GraphState, deps: AgentDeps) -> dict:
    payload = json.dumps({k: getattr(state, k).model_dump()
                          for k in ("visual","ux","accessibility","brand","market")})
    report = await deps.llm.complete(
        system=prompts.synthesizer_system(),
        user=payload,
        schema=DesignReport)
    violations = quality_gate.check(report)              # pure-Python checks
    if quality_gate.has_fail_severity(violations):
        report = await deps.llm.complete(                # 1-shot retry with violations annotated
            system=prompts.synthesizer_system(),
            user=payload + "\n\nQUALITY_GATE_FAILED: " + json.dumps(violations),
            schema=DesignReport)
    save_report(report, state.image_path)
    return {"report": report}
  16. 12 Report saved + UI renders A → E ▼

    The report lands at data/reports/<ts>-dashboard.json. The UI's "Report" tab populates: overall score, top strengths, top recommendations, accordion of findings, gallery of retrieved references.

Q: Is Image RAG a MCP? No. Image RAG is a retrieval technique (CLIP + LanceDB). MCP is a wire protocol. They are orthogonal — we use both. The search_designs MCP tool is one way the outside world reaches our Image RAG; the brand agent is another way the inside world reaches it. RAG sits at the bottom; both clients sit on top.

Hackathon vs production

Honest answer: this is a prototype. It is correctly factored to be made scalable, but several pieces would need to change before real traffic.

Hackathon-grade today

  • In-process LangGraph — fine for ~10 concurrent uploads.
  • Disk-backed JSON cache for LLM responses.
  • Embedded LanceDB in the app process.
  • Single-process Gradio with its built-in queue.
  • stdio MCP server — one client at a time.
  • No auth, no multi-tenancy.
  • All settings in one .env.

What would change for production scale (priority order)

  • Job queue — Celery+Redis or Temporal. Each upload becomes a durable task; UI subscribes via WebSocket/SSE.
  • Async everywhere — CI test ensures no sync IO leaks into agent paths.
  • Redis cache — replace disk cache; LRU eviction, TTL, hit-rate metrics.
  • Vector DB at scale — LanceDB on S3, or migrate to managed Pinecone/Qdrant once corpus exceeds 100k docs.
  • Rate limit + budget — leaky-bucket per user; per-tenant spend ceiling enforced inside cost.py.
  • Multi-tenancy — tenant_id column in LanceDB, per-tenant API keys, per-tenant LangSmith projects.
  • Auth + transport — swap stdio MCP for HTTP MCP behind a reverse proxy; OAuth / API tokens.
  • Structured logs — loguru → JSON to stdout, shipped via Vector/Fluent Bit to Loki or ELK.
  • Metrics + dashboards — Prometheus counters for agent latency, cache hit rate, eval pass rate, $/run; Grafana panels.
  • Horizontal scaling — agent workers are stateless; pull jobs from the queue; auto-scale on queue depth.
  • GPU pool for CLIP — batch image embeddings on a small GPU pool when ingest throughput grows; CPU is the fallback.
  • Eval as CI gate — the golden set runs on every PR; merges blocked if pass rate drops below threshold.
  • Cost ceilings + alerting — per-tenant budgets, Slack alert on overage.

Why this is a scoring win

  • Difficulty Score: this list proves we understand the production design space, not just hackathon code.
  • Code Quality: agents are pure (def run(state, deps)), the LLM lives behind a Protocol, IO is contained — each item above is a one-file swap, not a rewrite.
  • Concept Score: Sprint 5 cost-optimization concepts (caching, batching, tiering) are already in the design; production just pushes the same primitives into Redis instead of disk.
